# ADR-005：进程内实时态与扩展触发条件

- 状态：已接受
- 日期：2026-07-20

## 背景

当前产品以单机和可信局域网为主。`RoomHub` 的 SSE 队列、在线 presence、生成中的 task 和 in-flight 缓冲都在 FastAPI 进程内，使用 SQLite 作为持久化后端。

## 决策

1. 当前版本明确按单进程运行设计，`RoomHub` 和 `GenerationManager` 不承诺跨进程共享。
2. 应用必须在文档中说明重启会终止未完成生成，已落库事件仍可恢复。
3. 只有出现以下任一需求时才引入外部基础设施：多 worker/多副本、跨实例 SSE、生成任务可靠重试、长时间后台任务或需要独立扩缩容。
4. 触发扩展时采用 Redis Pub/Sub 或消息总线承载实时广播，采用持久化任务队列承载生成/解析任务，并增加 generation/job 状态表。
5. 在触发条件满足前，不提前引入微服务和 Redis，避免把桌面应用变成高运维系统。

## 后果

- 当前部署简单、启动快、离线可用。
- 进程崩溃会丢失内存订阅者和未完成 task，应用只能依赖已落库事件恢复。
- 迁移到多实例时需要同时迁移实时广播、任务状态、锁和可观测性，不能只替换 `RoomHub`。

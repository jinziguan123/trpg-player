# ADR-002：事件日志、序号与 SSE 重连

- 状态：已接受
- 日期：2026-07-20

## 背景

游戏界面同时需要低延迟流式叙事、离散规则事件、断线重连和历史分页。单纯依赖内存广播无法恢复，单纯依赖数据库轮询又无法提供流式体验。

## 决策

1. `event_logs` 是会话可重放事件的持久化来源。
2. `(session_id, sequence_num)` 必须由数据库唯一约束保证唯一；应用层分配序号发生冲突时回滚并重试。
3. SSE `/live` 只负责实时增量与当前生成缓冲，不承担长期可靠存储。
4. 断线恢复通过 `GET /events?before_seq=...` 拉取持久事件；前端按事件 id 幂等去重，按 `sequence_num` 排序。
5. 需要重排广播顺序时，先写入临时序号区间，再写入最终连续序号，避免唯一约束下的瞬时冲突。

## 后果

- 实时链路与可靠历史链路职责清晰，断线后可以恢复已落库内容。
- 流式 token 仍可能因断线无法逐 token 恢复，恢复粒度是最终持久事件；这是可以接受的协议边界。
- 序号分配仍是当前单体内的应用协调逻辑；若未来多实例写入，应迁移到数据库事务序列或独立事件存储。

# ADR-004：AI 语义裁定与规则确定性变更

- 状态：已接受
- 日期：2026-07-20

## 背景

LLM 适合理解玩家意图、生成叙事和提出场景判断，但不适合作为骰子、战斗、HP、SAN、物品和权限状态的最终事实来源。让自然语言直接改变状态会导致不可重放、重复副作用和规则漂移。

## 决策

1. AI 只产生结构化意图、TurnPlan、工具调用或叙事文本。
2. 骰子、检定、战斗、追逐、场景切换、道具和角色状态变更必须由确定性规则引擎或领域服务执行。
3. 所有确定性变更都必须通过事件日志和可审计状态写入落库。
4. planner、validator、摘要等增强调用可以 fail-open；核心状态守卫必须 fail-closed 或保证幂等。
5. 工具重试、生成取消和重新生成不能重复应用同一条状态副作用。

## 后果

- 核心玩法可以被测试、回放和重现，模型供应商变化不会直接改变规则结算。
- AI 编排代码需要维护计划、工具结果、事件顺序和回滚语义，复杂度高于单次 prompt。
- 新规则系统必须提供确定性引擎边界，不能只新增 prompt。

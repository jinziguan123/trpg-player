# ADR-001：桌面优先与可信局域网边界

- 状态：已接受
- 日期：2026-07-20

## 背景

TRPG Player 的主要数据是本地存档、用户上传的模组/素材和本地 AI 配置。当前部署由 Tauri 外壳启动 FastAPI sidecar，数据落在用户可写目录；多人能力面向可信局域网，而不是公网账号系统。

## 决策

1. 桌面单机是默认部署形态，Tauri + FastAPI sidecar + SQLite 作为首要运行闭环。
2. 局域网联机只定义为可信网络能力，`X-Player-Token` 是轻量席位归属标识，不等价于公网身份认证。
3. 当前版本不承诺公网安全，不提供账号、密码、TLS 终止、跨租户隔离、抗滥用和正式多租户能力。
4. 对外发布前必须完成内容/素材审计、代码签名和平台安装测试；不能因为使用 Apache-2.0 就推断规则书、模组、字体或模型权重可分发。

## 后果

- 可以保持本地优先、低运维和离线存档体验。
- CORS、明文 token 和 SQLite 不需要为了假想的云部署过早复杂化，但必须在启动文档和产品提示中明确边界。
- 若未来面向非可信网络，必须新增独立的身份、授权、传输安全和审计设计，不能只把当前 token 换个名字。

# ADR-003：`world_state` JSON 边界与版本

- 状态：已接受
- 日期：2026-07-20

## 背景

跑团包含剧情 flags、线索台账、NPC 记忆、战斗/追逐态、摘要、RAG 统计和 token 用量等不同变化频率的状态。全部拆表会增加早期开发成本，全部放在一个 JSON 中又会带来校验、迁移和并发更新问题。

## 决策

1. 保留 `GameSession.world_state` 作为剧情扩展状态和低频会话记忆容器。
2. 顶层键必须使用明确命名空间，并由 `schema_version` 标识整体版本。
3. 新代码通过 `world_state` 服务读写；禁止对 ORM 挂载的嵌套字典做原地修改后直接提交。
4. 战斗、追逐、回合锁、生成任务、用量统计等高频或强一致状态，随着需求稳定逐步拆成结构化表。
5. 每次 schema 变更都必须提供迁移函数或数据库迁移，并补充旧存档回归测试。

## 后果

- 可以继续快速支持剧情扩展，不需要为每个模组字段创建数据库列。
- JSON 查询、并发合并和字段兼容仍有成本，不能把 `world_state` 当成无限扩展的通用数据库。
- 拆表优先级由并发频率、查询需求和数据丢失风险决定，而不是按代码文件方便程度决定。

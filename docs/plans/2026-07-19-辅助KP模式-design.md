# 辅助 KP 模式（真人 KP + AI 参谋）设计稿

> 2026-07-19。核心翻转：系统从「AI 即 KP」变为「真人 KP 的驾驶舱」。
> 确定性引擎（骰子/战斗/台账/连通图/可见性）不区分模式；AI 从叙事者退位为参谋，
> 其输出**永不直接进入玩家可见流**——一切经真人 KP 之手采纳后发布。

## 1. 模式与席位

- `GameSession.kp_mode: "ai"（默认）| "human"`（迁移加列，旧局全为 ai，零影响）。
- human 模式必须有一个 **KP 席**：`SessionParticipant.role="kp"`，不绑角色卡、
  由创建者 token 认领（通常房主即 KP）。授权用统一的 `authorize_kp()`（与 GPT 正在做的
  authorize 系列合流）。
- 玩家视角完全不变：事件流、投骰、战斗面板、大地图照旧。

## 2. 权限翻转（human 模式下）

原 AI KP 独占的动作全部改由 KP 席触发，**复用既有确定性执行器，只换触发源**：

| KP 动作 | 复用的执行器 | 形态 |
|---|---|---|
| 发叙事 | add_event(narration) 直落直播 | KP 输入框，即时广播，不经生成管线 |
| NPC 台词 | npc_dialogue 事件 | 选 NPC 下拉 + 台词框（气泡带立绘） |
| 发起检定/对抗 | `_exec_dice_check` 挂待投骰 | 选人/技能/难度表单 |
| 场景切换 | `_exec_scene_change`（连通校验照跑） | 场景选择器 |
| 剧情标志 / 手书 / HP·SAN | `_exec_flag / _exec_handout / _exec_hp_change` | 按钮+表单 |
| 开战 | `_exec_start_combat`（战斗引擎全自动） | 选敌按钮 |

- 玩家 `/advance` 后**不再触发** `run_chat_generation`：改为广播「玩家回合已提交」
  提醒 KP。AI 队友回合保留为可开关项（KP 面板上「让 AI 队友行动」按钮/自动开关）。
- 暗骰结果（心理学等）与幕后推演事件改为呈现在 KP 面板（复用 kp-only visibility 哨兵，
  本就是为此设计的）。

## 3. AI 参谋面板（KP 专属侧栏，玩家不可见）

一切 AI 产出都是**草稿态**（不落库或落库带 `visibility=["kp"]`），KP 编辑后一键发布：

1. **叙事草稿**：「帮我写这段」→ 复用 `build_kp_context`（模组/真相/台账/记忆/RAG 全套）
   生成叙事草稿进面板 → KP 改后发布为 narration。
2. **裁定建议卡**：玩家回合提交后自动跑一次 planner（TurnPlan 从幕后决策者变成
   给真人的建议）：该不该检定、难度/奖惩骰、SAN 规格（吃场景 events 的模组数值）、
   线索该不该给、要不要开战——每项带「照此执行」按钮直连对应执行器。
3. **规则/模组速查**：RULE_LOOKUP / MODULE_LOOKUP 的 RAG 直接暴露成 KP 搜索框。
4. **NPC 台词候选**：选 NPC + 一句意图 → NPC agent 出 2-3 句候选，点选即发。
5. **自由生图**：KP 输入描述 → ComfyUI 出图 → 预览 → 发布为配图卡；四个自动配图
   接入点在 human 模式下改为「生成后先进 KP 面板预览再发布」。

## 4. 分期

- **M1 人肉 KP 工具桌**（最小可玩）：kp_mode 列 + KP 席 + 叙事输入/NPC 台词/
  发起检定/场景切换/手书按钮 + 关断自动生成链路 + 暗骰进 KP 面板。
- **M2 AI 参谋**：叙事草稿 + planner 建议卡 + 规则/模组速查。
- **M3 配图与队友**：KP 触发生图与预览发布、AI 队友回合开关。
- **M4 锦上添花**：KP 笔记本、导演信号（节奏提醒）面板化、语音转文字。

## 5. 设计原则

- **单一业务逻辑双触发源**：不为 human 模式复制任何执行器/规则逻辑，kp_mode 只决定
  「谁能扣扳机」与「AI 产物是草稿还是直播」。
- **AI 永不越过真人**：human 模式下 AI 无权直接写玩家可见事件（草稿/建议/kp-only 三态）。
- 与多人联机路线（真人玩家 + AI 队友）正交组合：真人 KP + 真人玩家 + AI 队友即完整
  线下团的线上形态。

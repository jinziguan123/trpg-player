# 战斗轮 & 追逐 设计文档（2026-07-10）

> 状态：**设计中（待评审）**。定稿后再落代码。
> 关键决策（已定）：战斗保真度 = 轻量结构化 C；NPC 决策 = 混合（杂兵引擎启发式 / 关键 NPC 子代理）；
> 战斗行动 UI = 目标/武器选择器 + 闪避/逃跑按钮；先攻 = **火器优先** + DEX；追逐 = 抽象距离轨；
> 战斗结束 = 引擎自动判 + 子代理可主动收；**战斗上下文独立于主 KP 上下文**，KP 只在战斗结束后读结果。

---

## 1. 背景与目标

现状：战斗完全靠 KP 提示词即兴驱动（`kp_system.py` 战斗段三行），引擎只提供零件
（`opposed_check` 对抗、`hp_change` 含自动重伤、106 件武器表、DB/MOV/闪避派生）。**缺"轮"的骨架**——
无先攻队列/回合追踪/行动经济/多打多管理/武器伤害自动化，导致：丢先攻顺序、一轮动两次、伤害靠 LLM 瞎编、
多打多失控。追逐仅一句提示，无状态。

目标：把**战斗轮**做成一个**半独立子系统**——有明确 start/end、自己的精瘦上下文、确定性引擎结算、
中途可加敌/友、结束产出结构化结果折回主线。**追逐**做成**抽象距离轨**子系统，与战斗在"逃跑"处咬合。

## 2. 设计原则

1. **上下文独立（核心）**：战斗子系统用自己的**精瘦上下文**（参战方战斗数值 + 战斗态 + 眼前情境），
   **不**带主 KP 的世界记忆/线索台账/NPC 秘密/模组 RAG。主 KP **只在战斗结束后读一份结果摘要**。
   理由：战斗结算与"回忆线索"无关；小上下文更便宜、更不跑偏；主线叙事保持干净。
2. **引擎管确定性，LLM 管叙事+决策**：先攻序、命中判定、武器伤害、HP/重伤/死亡、闪避/反击、弹药、
   大失败/故障——全走 `RuleEngine`（复用现有零件）。LLM 只做：NPC 战术决策（关键 NPC）+ 逐轮叙述 + 结束摘要。
3. **战斗是一段自包含插曲**：逐轮血肉照常落库给玩家看，但**只有摘要进主 KP 大上下文**。
4. **复用而非新造**：对抗走现有 `_resolve_opposed`；伤害结算走 `_exec_hp_change`（已含自动重伤/濒死/死亡）；
   武器数据取 `COC_WEAPONS`；闪避/DB/MOV 取角色派生值。

## 3. 战斗状态 schema（`world_state.combat`）

独立于叙事，落在会话 `world_state`：

```jsonc
{
  "active": true,
  "round": 3,
  "turn_index": 1,                 // 当前轮到先攻序里的第几个
  "initiative": [                  // 先攻序（每轮开始时重排，见 §4）
    {
      "id": "npc_corbitt",         // 角色 id 或 NPC id
      "name": "沃尔特·科比特",
      "side": "enemy",             // player | ally | enemy
      "dex": 65,
      "has_firearm": false,        // 本轮是否持已上膛火器（决定火器优先）
      "hp": 12, "max_hp": 13,
      "status": "ok",              // ok | major_wound | dying | dead | fled
      "weapon": "徒手格斗",
      "skills": {"格斗(斗殴)": 45, "闪避": 30, "射击(手枪)": 0},
      "db": "0",                   // 伤害加值
      "mov": 7,
      "acted_this_round": false,
      "dodged_this_round": false   // 已用闪避/反击占用本轮行动的标记
    }
  ],
  "log": [ {"round":1,"actor":"...","action":"attack","target":"...","result":"...","hp_delta":-5} ],
  "flee_to_chase": null            // 逃跑触发时挂载的追逐引用（见 §11）
}
```

参战方数值来源：玩家/队友取 `Character`（skills/system_data/派生 DB/MOV/闪避）；NPC 取 `module.npcs`
的 attributes/skills（解析时已生成）。杂兵可只带必要战斗字段。

## 4. 先攻规则（火器优先 + DEX）

**每轮开始重排先攻序**（因为持火器状态、加入/退出会变）：

1. 先按"**是否持已上膛火器**"分两组：火器组在前、近战/其它组在后。
2. 组内按 **DEX 降序**。
3. **平手**（同组同 DEX）：先比相关战斗技能高者；仍平则玩家先于 NPC；再平用确定性 tie-break（按 id 稳定）。

说明：这是 CoC 7e "火器可在同 DEX 抢先"的**可测简化**（火器组整体先于近战组，而非逐人插空）。
RAW 的"抽枪/瞄准调整、连射次数"留作后续增量。**弹药/装弹/故障率**（`COC_WEAPONS.err`）纳入攻击结算但
先做最小版（打空需装弹占一动；掷出 ≥err 判故障）。

## 5. 生命周期与回合流转

战斗是会话的一个**模式**（`combat.active`），期间**主 KP 叙事循环挂起**，改由战斗子系统驱动。

- **start_combat(participants, trigger)**：主 KP tool-loop 里 KP 判定开打时调用（或引擎侦测到攻击动作）。
  引擎拉各方数值、算首轮先攻、round=1、广播"进入战斗"、**多人回合从确认制切先攻队列**（见 §9）。
- **回合推进**（交互式，跨多次玩家输入，类比主游戏）：
  从 turn_index 起沿先攻序推进——
  - 轮到 **NPC/敌/AI 队友** → §7 决策 → 引擎结算 → 子代理叙述这一动 → turn_index++。
  - 轮到 **玩家** → **暂停**，前端提示**该玩家**声明战斗行动（§10）。玩家提交 → 引擎结算 → 子代理叙述 → 继续。
  一轮走完 → round++、重排先攻、清 acted/dodged 标记。
- **add_combatant(spec)**：中途加敌/友，按 §4 插入当前先攻序（本轮或下轮加入，由触发时机定）。
- **end_combat**：满足结束条件（§见下）→ 引擎产出**结果摘要**（§12）→ 退出战斗模式、回主 KP 叙事。
  **结束判定（双管）**：引擎自动——某一方全部 dead/dying/fled 即终；子代理/KP 也可主动收
  （敌投降/撤退、玩家脱离）。

## 6. 动作解析（复用现有零件）

行动类型：`attack | dodge | fight_back | flee | other`（other 走普通检定/自由叙述）。

- **attack（近战/远程）**：
  - 命中：近战 = 攻方"格斗(X)" **对抗** 守方选择的应对（闪避/反击，见下）；远程 = 攻方"射击(X)"检定
    （距离/掩体调难度），守方通常不能闪避子弹（可趴下/找掩体作情境调整）。走 `_resolve_opposed` / `_exec_dice_check`。
  - 伤害：命中后**引擎从 `COC_WEAPONS` 取该武器 `dam` 公式**（如 `1D8+DB`、`1D6+半DB`），代入攻方 DB
    掷出 → `_exec_hp_change(target, -伤害)`（**已含自动重伤体质检定/濒死/死亡**）。**LLM 不再报伤害数字**。
  - 贯穿武器（`tho=1`）大成功时伤害骰翻倍等细则纳入引擎。
- **dodge（闪避）**：守方用"闪避"对抗攻方命中；**占用该守方本轮行动**（`dodged_this_round`）。
- **fight_back（反击）**：守方用"格斗"对抗；胜则反造成伤害；同样占本轮行动。
- **flee（逃跑）**：脱离接触 → 触发 §11 追逐距离轨；成功拉开则该参战方 status=fled、移出先攻序。
- 大失败（fumble）/武器故障：按 CoC 表（掉武器/卡壳/误伤）由引擎给结果、子代理叙述。

## 7. NPC 决策（混合）

- **杂兵/怪物 → 引擎启发式**（零 LLM，省钱防乱）：可配策略如"攻击最近/最弱/威胁最大的敌方"、
  "HP<25% 且非狂暴则撤退"。策略参数可放 NPC 数据（`combat_ai: aggressive|cautious|beast…`）。
- **关键 NPC（有 personality/secrets）→ 战斗子代理**（LLM，精瘦上下文）：做**战术决策 + 台词/叙述**，
  保持性格（如科比特会威胁、拖延、施法）。每个这类 NPC 回合一次调用。

## 8. 战斗子代理上下文（独立、精瘦）

`CombatAgent`（新，仿 `NPCAgent`/`backstage_agent`）。系统提示 = **战斗专用**（CoC 战斗裁定摘要 + 叙述纪律：
不替玩家决定、不报伤害数字、按引擎结果叙述）。注入内容仅：**当前 combat 状态**（先攻序/回合/各方 HP/status）、
**本动作的引擎结算结果**、**眼前场景一句话**、（关键 NPC 决策时）**该 NPC 的 personality/动机**。
**不注入**世界记忆/线索/RAG/模组全文。

## 9. 多人回合模型切换（确认制 → 先攻队列）

战斗期间**覆盖**现有"全员确认→整批推进"（`session_service.turn_confirm_*`）：

- 非战斗：维持确认制（不变）。
- 战斗：改为**先攻队列**——只有"轮到的那个玩家"能提交战斗行动，其余玩家等待；NPC 回合系统自动推进。
- 前端明确展示："战斗 · 第 3 轮 · 先攻：A→B→C · 轮到你/轮到 A"。掉线玩家轮到时：超时或房主"替其防御/跳过"
  （复用 §我们已加的掉线豁免思路），避免卡死。

## 10. 前端战斗 UI

轮到本玩家时弹**战斗行动面板**：
- **目标选择器**：当前 side≠己方 的存活参战方下拉。
- **武器选择器**：该角色可用武器（角色卡武器/装备 + "徒手" + 场景可取物），带命中技能与伤害预览。
- **闪避 / 反击 / 逃跑 按钮** + "其它行动"文本框（自由声明，走 other）。
提交后经 `POST /combat/action`。非战斗回合面板隐藏。战斗态用独立视觉（先攻条 + 各方 HP 条 + 回合数）。

## 11. 逃跑 → 追逐（抽象距离轨）

独立子系统 `world_state.chase`：
```jsonc
{ "active": true, "round": 2, "gap": 3,          // 领先/差距的数值（正=逃方领先，越大越可能脱身）
  "participants": [{"id","name","side":"quarry|pursuer","mov":8}],
  "escape_at": 6, "caught_at": -3,               // 越过阈值判脱身/被追上
  "hazards": [] }
```
每轮：双方按 **MOV 调整**做对抗（体质/运动/驾驶等，视载具）→ 胜负推动 `gap` ±；环境障碍 = 额外检定
（失败被绊/受伤）。`gap≥escape_at` 判脱身、`gap≤caught_at` 判被追上（可回落战斗）。战斗 flee 成功即
`start_chase(...)`；追逐结束同样产出摘要折回主 KP。

## 12. 结果折回主 KP

`end_combat`/`end_chase` 产出**结构化摘要**（存 `world_state`，注入下一次主 KP 上下文）：
`{outcome:玩家胜/败/逃/敌逃, casualties:[{name,status}], hp_after:{...}, notable:["科比特被击倒但尸身消失…"]}`。
主 KP 读摘要续写余波（不重放逐轮血肉）。逐轮事件已落库供玩家回看/战报。

## 13. 工具 / API 面

## 14. 检验（pytest + evals 分工）

- **确定性引擎 → pytest（必须、可完全确定性测）**：
  先攻排序（火器组先 + DEX 降序 + 平手 tie-break）；武器伤害 = 表 `dam` 骰 + DB（含半DB、贯穿翻倍）；
  命中/闪避/反击对抗成功等级；重伤/濒死/死亡阈值（复用现有）；弹药耗尽需装弹、`err` 故障；加/移除参战方
  重排先攻；结束判定（一方全 dead/fled）；逃跑→距离轨移交；距离轨 gap 推动与阈值判定；启发式 NPC 选目标。
- **子代理编排 → evals（judge）**：新 fixture + 评分项：
  `combat_turn_order`（守先攻序、不乱序/不重复行动）、`combat_damage_from_engine`（伤害只由引擎给、不瞎编）、
  `downed_cannot_act`（昏迷/死亡者不再行动）、`combat_end_summary_faithful`（摘要与逐轮一致）、
  `chase_distance_consistency`（距离更新自洽）、`no_player_impersonation_in_combat`（不替玩家决定攻击/防御）。
- **状态机不变量 → pytest**：拒绝非当前回合玩家的行动；回合/轮推进正确；掉线玩家不卡死。

## 16. 风险与未决

- **LLM 破协议**：子代理仍可能不守流程（deepseek 前科）。缓解：引擎强约束（先攻序/伤害/结束由引擎定，子代理只叙述），
  子代理越权（替玩家、瞎报伤害）由引擎兜底忽略 + evals 盯。
- **交互式战斗跨多次输入的状态一致性**：战斗态存 `world_state` 落库，断线重连按 §我们已修的 ready/权威态恢复。
- **先攻火器规则的 RAW 取舍**：先做"火器组整体优先"简化；连射/抽枪/瞄准留 P 后。
- **杂兵 vs 关键 NPC 的判定**：默认有 personality/secrets 的走子代理，其余启发式；可在 NPC 数据显式标 `combat_ai`。
- **战斗与"分头行动"叠加**：v1 先假设战斗时全队同场（不与分头并发）；分头中开打的编排留后。
- **UI 复杂度**：目标/武器选择器需要角色可用武器数据；若角色卡未列武器，先给"徒手 + 常见 + 自由填"。

# 战斗轮 P2：主动动作集 实施计划

> 承 [2026-07-11-战斗轮重构-design.md](2026-07-11-战斗轮重构-design.md) §5。P1（可暂停状态机 + 反应 + 状态层）已合入 master。本阶段补主动动作集。**直接在 master 开发**（用户已授权）。

**Goal:** 战斗自己回合除「攻击/逃跑」外，支持 急救·医学 / 观察战场 / 擒抱·缴械 / 装填·瞄准。

**Architecture:** 规则判断进纯 `app/rules/coc/combat.py`（可 monkeypatch 钉死骰子、pytest 全覆盖）；落库/广播/状态机在 `combat_service.py`；端点 `/combat/action` 已能收 `type`，扩 schema 支持新字段；前端 `CombatPanel` 加动作按钮。擒抱/缴械引入**正交条件** `conditions: list[str]`（与既有 `status` 并存）。

**通用纪律:** TDD；后端 commit 前 `cd server && .venv/bin/pytest -q && .venv/bin/ruff check .` 全绿；前端 `cd apps/web && npx tsc --noEmit && npx vite build`；禁 emoji、图标 `react-icons/gi`；commit 中文 + `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`。

## 数据模型增量

参战方 dict 新增（`_char_participant`/`_npc_participant` 默认给空/False）：
- `conditions: list[str]` —— 正交条件，元素 ∈ `{"grappled","disarmed"}`（与 `status` 的 ok/major_wound/dying 并存）。
- `aim: bool` —— 上一动是否瞄准（下一击加一个奖励骰）。
- `first_aid_used: bool` —— 本处伤是否已被急救过（每处伤一次，RAW）。

## Part A — 引擎纯函数（`combat.py` + `tests/test_combat.py`）

### A1 `resolve_first_aid(medic_data, skill="急救") -> dict`
急救 or 医学检定。返回 `{success, heal, lines}`：成功 `heal=1`（急救回 1 HP；医学可后续加成，先同 1），失败 `heal=0`。落库时由 service 决定：目标濒死则成功=稳住（status dying→unconscious，稳定但出局）+ 该处伤标记已急救。

### A2 `resolve_observe(observer_data, skill="侦查") -> dict`
侦查 or 心理学检定。返回 `{success, lines}`：成功产一条「察觉到破绽/敌情」beat（不改数值，交子代理叙述）。

### A3 `resolve_maneuver(attacker_data, defender_data, kind="grapple") -> dict`
对抗格斗（attacker 的格斗 vs defender 的格斗/闪避，用 `compare_checks`）。返回 `{success, condition, lines}`：attacker 胜则 `success=True`、`condition = "grappled" if kind=="grapple" else "disarmed"`；否则 success=False、condition=None。不造成伤害。

### A4 条件对攻防的影响（改 `resolve_attack` + `allowed_reactions`）
- `allowed_reactions(is_firearm, defender_grappled=False)`：被擒抱者**不能闪避**（也不能扑掩体）——近战时 allowed 只剩 `["fight_back"]`；火器时 grappled 则 allowed 为 `[]`（无从躲避，命中即结算）。默认参数保持向后兼容。
- `resolve_attack` 增可选入参 `attacker_disarmed: bool = False`：为真时武器强制 `徒手格斗`（缴械后只能徒手），其余逻辑不变。

## Part B — 状态机 + API（`combat_service.py`、`schemas/combat.py`、`api/combat.py`）

### B1 `_apply_one_action` 动作分发
`atype` 新增分支（在 attack 之前/之后）：
- `first_aid`：`target_id` 指向己方受伤者；调 `resolve_first_aid`，成功 → `apply_heal(target, heal)`（新小工具，回血写回角色卡，配 HP 上限）+ 濒死稳住 + `target["first_aid_used"]=True`；产 beat/日志行。
- `observe`：调 `resolve_observe`；成功 → beat 交子代理叙述。
- `maneuver`：`target_id` 敌方；`kind` 来自 action；调 `resolve_maneuver`，成功 → `target["conditions"].append(condition)`（去重）；产日志行。
- `reload`：`actor` 火器 → 标记 `loaded`（占一动）；产日志行。
- `aim`：`actor["aim"]=True`（占一动）；产日志行。
- attack 分支：消费 `actor.get("aim")`（有则 `resolve_attack(..., bonus? )`——注：`resolve_skill_check` 支持奖惩骰，attack 命中检定加 1 奖励骰后清 `aim`）；消费 `actor` 的 `disarmed` 条件（传 `attacker_disarmed=True`）。
- 攻击真人时的暂停（drive_npcs）与 `resolve_reaction`：`allowed` 计算带上 defender 的 `grappled`（近战被擒抱只能反击）。

### B2 schema
`CombatActionRequest` 增 `kind: str | None = None`（擒抱/缴械用：`grapple|disarm`）；`target_id` 复用（first_aid 指己方、maneuver 指敌方）。

### B3 API
`/combat/action` 已透传 `data.model_dump(exclude_none=True)` 给 `resolve_player_action`→`_apply_one_action`，新字段自动带上，无需改端点签名。补端点/服务测试覆盖各新动作。

## Part C — 前端（`CombatPanel.tsx`）
自己回合的动作栏在「攻击/闪避/逃跑」旁增按钮（`react-icons/gi`）：
- 急救 `GiHealthNormal`（选己方受伤者为 target）、观察 `GiEye`、擒抱 `GiGrab`+缴械（kind 选择或两个按钮 `GiGrab`/`GiWhiplash`）、装填 `GiReloadGunBarrel`、瞄准 `GiEyeTarget`（图标名以 gi 实际存在为准，不存在则换近义 gi 图标）。
- 提交走既有 `submit({type, target_id?, kind?})`。目标选择器按动作切换己方/敌方候选。
- 参战方卡片显示 `conditions`（擒抱/缴械）小徽标。

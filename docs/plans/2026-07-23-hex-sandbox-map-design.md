# 大地图二形态：六边形沙盘 + 战争迷雾

**背景与动机**：KP（AI）对场景之间的地理位置掌握不够——叙述方向、距离、途经时没有一致的
参照系，玩家侧也缺「这张地图上我在哪、哪边还没去过」的空间感。用户拍板做文明式沙盘：
hex 网格 + 标记表达场景间位置关系，带战争迷雾（玩家只见已知场景；KP 可切上帝/玩家视角），
美术不能简陋且要适应森林/海洋/沙漠/城市等风格。

**已确认的方向**（2026-07-23 与用户对齐，四项均按推荐方案）：
- 布局模型：**六边形网格**（axial 坐标；方位/距离可确定性计算，直接喂 KP 上下文）
- 落位方式：**AI 解析提议 + 确定性校验修复 + KP 拖拽修正**
- 美术路线：**程序化矢量瓦片**（konva 程序纹理 + gi 图标群落，全走 CSS 变量，零素材管线）
- 与调查板：**双视图并存**（大地图弹窗页签：沙盘管地理迷雾，调查板管线索关系网）

---

## 一、红线与第一性原理

**这不是被删的瓦片图。** 2026-07-04 删掉的 MapView 是场景**内部**几何（房间/墙/门），
模组文本对那一层确实欠定。本设计只做场景**之间**的相对位置，而这一层模组文本有充分语义
（「镇北的教堂」「沿河的仓库」「六号车厢连着七号」），且已有结构化的 `connections` 邻接图兜底。

**坐标是象征性相对位置，不是测绘。** 一格不代表固定公里数；系统只承诺两件事：
**方位**（谁在谁北边）和**相对远近**（紧邻/不远/较远）。不承诺比例尺、不做格间移动力、
不做 hex 寻路——travel 仍走 `connections` 图的 BFS 校验，沙盘不改变移动规则。
这就是绕开「几何欠定」的方式：欠定的是精确几何，我们只取文本足以支撑的那一层语义，
再加 KP 拖拽做人工兜底。

**可复用资产**（调研确认全部现成）：
- 拓扑：`scene.connections` + `_scene_adjacency`/`find_scene_path`（session_service）
- 迷雾：`visited_scenes` + `known_scene_ids`（已含关键词解锁「听说过」态）+
  `list_known_locations`（已产出过滤后的前端 payload：connections/队友/线索）
- 隔离：`visibility=["kp"]` 哨兵、KP 身份判定（`kp_mode`）
- 渲染：`react-konva@19` 已在依赖里；哥特暗色/羊皮纸双主题 CSS token；调查板的烛光板面

## 二、数据模型

`Module.scenes[i]` 增加可选字段（存量模组无此字段 → 确定性回填，见 §三）：

```json
"map": {"q": 0, "r": -2, "biome": "urban"}
```

- `q/r`：axial 六边形坐标（pointy-top），整数，location 场景全图唯一；
  `kind=="chapter"` 场景无 map（不上图，与调查板同一规则）。
- `biome`：地貌枚举，渲染与叙事共用：
  `forest / water / coast / desert / mountain / plain / swamp / urban / ruin / interior`。
  室内型场景（车厢、宅邸房间）用 `interior`——hex 沙盘对纯室内模组照样成立，
  格子表达的是房间/车厢的相对位置关系。

## 三、落位管线（AI 提议 → 确定性修复 → KP 修正）

1. **解析提议**：`PARSE_PROMPT_TEMPLATE` 场景结构里加 `map` 字段与规则条目——按文本
   方位语义提议坐标（北在 -r 方向），相连场景距离 1–3 格，坐标不重叠，按环境给 biome。
2. **确定性校验与修复**（新 `app/services/hex_map.py`，解析完成后立即执行）：
   - 硬规则：坐标必须唯一整数；违规/缺失的场景重新落位。
   - 落位算法（确定性，同输入同输出）：按「已定位邻居的重心就近 + 螺旋找空位」逐个放置，
     顺序按连通图 BFS（起点=度数最高的已定位场景，全缺则第一个 location 场景放原点）。
     LLM 的合法提议全部保留——修复只补洞，不推翻。
   - 软规则：相连场景距离 > 6 只告警不强改（尊重「遥远但直达」的语义，如传送/长桥）。
3. **存量回填**：老模组首次触达时（`/locations` 端点、解析入口）跑同一修复器懒回填并落库，
   幂等；回填纯拓扑布局，KP 可再拖。
4. **KP 修正**（P-Hex-4）：编辑模式拖场景到新格，`PATCH /modules/{id}/scene-map`
   （房主/真人 KP），后端仍过唯一性校验。

## 四、KP 空间语义注入（本需求的原始动机，最先落地）

坐标存在后，方位与远近可**确定性**算出（`hex_map.direction_word` 八方位 +
`distance_word` 远近档），注入两处：

- `context.py` 的【场景连通】段升级：
  「由此可直达：图书馆（北・紧邻）、旧仓库（东南・不远）」；
- 当前场景补一句地貌：「当前场景地貌：城镇边缘的密林」（biome 中文名）。

KP 叙述方向、来路、途经从此有一致参照系；planner 的场景裁定同样受益（同一份上下文）。
无坐标（回填未触发）时该段维持现状——fail-open，不阻塞任何既有流程。

## 五、视角与迷雾（P-Hex-2）

`GET /sessions/{id}/locations` 现状对所有人跑同一套迷雾，无 KP 分支。改造：

- 玩家（默认）：维持现状，known 场景才返回，附 `map` 字段。
- 真人 KP（`kp_mode=="human"` 且请求者是 KP）：返回**全部** location 场景，每条附
  `known/visited` 标记——前端「玩家视角」toggle 纯客户端过滤，不用二次请求。
- 迷雾三态（前端渲染语义，数据已齐）：
  已到访=全彩完整；听说过未到访（keyword 解锁）=剪影+名称（去饱和、虚线描边）；
  未知=迷雾覆盖（KP 上帝视角下则以「玩家未知」样式提示）。

## 六、沙盘渲染（P-Hex-3，react-konva）

- **坐标系**：axial→pixel（pointy-top），Stage 支持平移/缩放（滚轮+拖拽），初始视野
  = 已知场景包围盒 + 边距。
- **地形场**：场景格实心地貌瓦片；其周围 1 圈空格做同 biome 的淡出晕染（大陆感），
  其余空白为羊皮纸/暗色虚空肌理。每 biome 一套「底色渐变 + 程序纹理 + gi 图标群落」：
  森林=层叠松杉、水域=横向波纹、沙漠=沙丘点画、城市=屋顶剪影群、山地=峰线、
  室内=石板/木纹。颜色全部运行时读 CSS 变量（getComputedStyle），双主题自适应。
- **标记层**：场景名牌（羊皮纸小签）、当前位置烛光光晕（呼吸动效）、队友徽章
  （SeatIcon 语义）、线索数徽记、danger 色环。图标一律 gi/lucide，禁 emoji。
- **迷雾层**：未知区域暗雾纹理 + 软边缘；「听说过」格半透剪影。
- **交互**：点已知场景 → 复用 confirmTravel → `/travel`（含二次确认）；
  KP 视角 toggle；编辑模式拖拽（P-Hex-4）。
- **入口**：GameSessionPage 大地图弹窗改双页签「沙盘 | 调查板」，调查板原样保留。

## 七、分期与验收

| 期 | 内容 | 验收 |
|---|---|---|
| P-Hex-1 | 数据模型 + 解析提议 + 确定性修复/回填 + KP 空间语义注入 | pytest 全过；evals --smoke 过；hex_map 单测（距离/方位/修复确定性/幂等） |
| P-Hex-2 | /locations KP 上帝视角分支 + payload 带 map + PATCH 场景坐标端点 | pytest 全过；玩家侧迷雾行为不变（既有断言不破） |
| P-Hex-3 | konva 沙盘：地形场/迷雾三态/标记/travel 交互 + 双页签 | tsc + vite build 过；浏览器验证双主题、三态迷雾、点击前往 |
| P-Hex-4 | KP 编辑拖拽 + 玩家/上帝视角切换打磨 | 浏览器验证；拖拽落库幂等 |

**不做**：场景内部几何（红线）；比例尺与格间移动力；hex 寻路（travel 仍走 connections）；
大场景占多格（v2 再议）；AI 生图底图（搁置，与调查板结论一致）。

**硬约束**：图标一律 `react-icons/gi` / `lucide-react` 矢量，禁 emoji；后端不把图标塞进
内容字符串；改 KP 上下文段落的提交按约定跑 evals 对比。
